## Contributions to the User Guide

### Listing all trips : `list`
Shows a list of all trips currently in the log and displays a **Summary Dashboard** in the result box. The list can be optionally sorted by a specific key.

The **Summary Dashboard** categorizes your trips based on the current date:
* **Upcoming**: Trips starting after today.
* **Ongoing**: Trips currently in progress.
* **Completed**: Trips that have already ended.
* **Planning**: Trips with no start date specified.

Format: `list [sort/KEY]`  
Example: `list sort/len` — Displays all trips ordered by duration (longest first).

## Contributions to the Developer Guide

### Trip Statistics and Multi-Key Sorting
The `ListCommand` bridges the `Logic` and `Model` layers to transform a simple list view into a temporal dashboard.

**Sorting Mechanism & Tie-Breaking:**
Sorting is implemented using a **Comparator Factory** pattern within `ListCommand`. To ensure a stable user experience, a **Multi-Level Tie-Breaker** is used:
1. **Primary Key**: The user-selected key (e.g., `start date`).
2. **Secondary Key**: A case-insensitive alphabetical sort by trip name retrieved via `Trip#getNameLowerCase()`.

**Temporal Dashboard Logic:**
Categorization logic is centralized in `TripSummaryUtil#calculateSummary()`. It determines status based on `LocalDate.now()`:
* **Planning**: `startDate == null`
* **Upcoming**: `today < startDate`
* **Completed**: `today > endDate`
* **Ongoing**: `startDate <= today <= endDate`
